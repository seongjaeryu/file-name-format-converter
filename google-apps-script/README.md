# Google Drive 한글 파일명 자동 변환 (Apps Script)

&nbsp;

> Mac에서 Google Drive에 올린 파일의 한글 이름이 깨져 보이나요?
>
> 이 스크립트를 설정하면 **자동으로 파일명을 수정**해줍니다.
> 한 번 설정하면 이후에는 아무것도 할 필요가 없습니다.

&nbsp;

---

&nbsp;

## 이 스크립트가 하는 일

Mac에서 만든 파일은 한글 이름이 `ㅎㅏㄴㄱㅡㄹ.txt` 형태(NFD)로 저장됩니다.

이 스크립트는 Google Drive의 지정된 폴더를 주기적으로 확인하여,
깨진 파일명을 `한글.txt` 형태(NFC)로 **자동 변환**합니다.

- 파일 내용은 전혀 변경되지 않습니다 (이름만 변경)
- 변환 내역은 Google Sheet에 자동으로 기록됩니다

## 설정 방법 (10분)

&nbsp;

### 1단계: Google Sheet 만들기

1. [Google Sheets](https://sheets.google.com)에서 **새 스프레드시트**를 만듭니다.

2. 이름을 `한글 파일명 자동 변환` 등 원하는 이름으로 변경합니다.

&nbsp;

### 2단계: Apps Script 열기

1. 스프레드시트 상단 메뉴에서 **확장 프로그램 → Apps Script**를 클릭합니다.

2. 새 탭에서 Apps Script 편집기가 열립니다.

&nbsp;

### 3단계: 코드 복사하기

1. 편집기에 기본으로 있는 코드(`function myFunction()...`)를 **모두 삭제**합니다.

2. [Code.gs](./Code.gs) 파일의 내용을 **전체 복사**하여 편집기에 붙여넣습니다.

3. 왼쪽 상단의 💾 저장 버튼을 클릭합니다 (또는 Ctrl+S / Cmd+S).

&nbsp;

### 4단계: Advanced Drive Service 활성화

1. Apps Script 편집기 왼쪽 메뉴에서 **⚙️ 프로젝트 설정**을 클릭합니다.

2. 아래로 스크롤하여 **`appsscript.json` 매니페스트 파일을 편집기에 표시** 옵션을 켭니다.

3. 왼쪽 파일 목록에서 `appsscript.json`을 클릭합니다.

4. 기존 내용을 모두 삭제하고 [appsscript.json](./appsscript.json) 파일의 내용을 붙여넣습니다.

5. 💾 저장합니다.

&nbsp;

### 5단계: 처음 실행하기 (설정 시트 생성)

1. 스프레드시트로 돌아가서 **페이지를 새로고침**합니다.

2. 상단 메뉴에 **파일명 변환** 메뉴가 나타납니다.

3. **파일명 변환 → 지금 실행**을 클릭합니다.

4. **권한 요청 화면이 나타납니다** — 아래 안내를 따라주세요.

5. 권한 승인 후, "설정 시트에 폴더 ID를 입력해주세요" 안내가 나타납니다. 이것이 정상입니다 — 6단계에서 설정합니다.

&nbsp;

### 6단계: 대상 폴더 ID 설정

1. Google Drive에서 변환할 파일이 있는 **폴더**를 엽니다.

2. 브라우저 주소창에서 폴더 ID를 복사합니다:

   ```text
   https://drive.google.com/drive/folders/1ABCdefGHIjklMNOpqrSTUvwxYZ
                                          ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                          이 부분이 폴더 ID입니다
   ```

3. 스프레드시트에 자동 생성된 `설정` 시트를 엽니다.

4. A2 셀에 폴더 ID를 붙여넣습니다. 여러 폴더를 감시하려면 A3, A4... 에 추가합니다.

5. **파일명 변환 → 지금 실행**을 다시 클릭하면 변환이 시작됩니다!

&nbsp;

### ⚠️ "앱이 확인되지 않았습니다" 경고

처음 실행할 때 Google이 아래와 같은 경고를 표시합니다. **정상적인 절차입니다.**

> 이 스크립트는 본인이 직접 만든 것이므로 안전합니다.

1. "이 앱은 Google의 확인을 받지 않았습니다" 화면이 나타나면

2. 왼쪽 하단의 **"고급"**을 클릭합니다.

3. **"(프로젝트 이름)(으)로 이동 (안전하지 않음)"**을 클릭합니다.

4. 권한 목록을 확인하고 **"허용"**을 클릭합니다.

&nbsp;

> 요청되는 권한:
> - **Google Drive 파일 보기 및 관리**: 파일 이름을 변경하기 위해 필요합니다.
> - **스프레드시트 보기 및 관리**: 설정과 변환 로그를 기록하기 위해 필요합니다.

&nbsp;

### 7단계: 자동 실행 설정 (선택)

파일이 올라올 때마다 수동으로 실행하지 않아도 되게 하려면:

1. **파일명 변환 → 15분 자동 트리거 켜기**를 클릭합니다.

2. 이후 15분마다 자동으로 새로 올라온 파일을 확인하고 변환합니다.

3. 자동 실행을 끄려면: **파일명 변환 → 자동 트리거 끄기**

## 동작 확인

- **변환 로그** 시트에서 변환된 파일 목록을 확인할 수 있습니다.

- 각 행에 시각, 파일 ID, 원본 파일명, 변환 후 파일명, 폴더 경로가 기록됩니다.

- 변환이 필요한 파일이 없으면 로그에 아무것도 추가되지 않습니다 (정상).

## 문제 해결

&nbsp;

| 증상 | 해결 방법 |
| --- | --- |
| "파일명 변환" 메뉴가 안 보임 | 스프레드시트를 새로고침 (Ctrl+R / Cmd+R) |
| "설정" 시트가 없음 | "파일명 변환 → 지금 실행"을 한 번 실행하면 자동 생성됩니다 |
| "폴더 접근 실패" 로그 | 폴더 ID가 올바른지, 해당 폴더에 접근 권한이 있는지 확인 |
| 변환이 안 됨 | 공유 드라이브의 경우 "편집자" 이상의 권한이 필요합니다 |
| 파일이 너무 많아 한 번에 안 끝남 | 정상입니다. 15분 트리거를 켜두면 자동으로 이어서 처리합니다 |

## 기술 참고

- V8 런타임의 `String.prototype.normalize('NFC')`를 사용합니다.
- 최초 실행: `DriveApp.getFiles()`로 전체 스캔 (continuation token으로 배치 처리)
- 이후: Drive API `changes.list`로 변경된 파일만 증분 처리 (API 쿼터 절약)
- `LockService`로 중복 실행을 방지합니다.
- Google Drive는 동일 폴더 내 동일 파일명을 허용하므로 이름 충돌 처리가 필요 없습니다.
